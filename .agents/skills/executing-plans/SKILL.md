# 执行计划

这是可选的执行类技能。

不属于默认工作流。

仅当用户明确要求在当前会话中按文档驱动执行时使用。

## 可接受的输入

本技能可从以下任一文档执行：

- `plan.md`
- `prd.md`
- 用户明确拖入或指向的任务文档

## 文档处理规则

### 若文档已可直接执行

则从中直接执行。

### 若文档为简单的 `prd.md`

可从该文档推导最小执行清单并直接执行。

不必强行先单独写 `plan.md`。

### 若文档为复杂的 `prd.md`

不要猜测。

停止直接执行，先使用本地 `writing-plans` 工作流创建：

- `.trellis/tasks/<task>/plan.md`

## 执行边界

若仅阅读并分类文档，仍属分析阶段。

若即将开始改代码，必须先做等价于：

- `$before-dev`

代码改动结束后，必须再做等价于：

- `$check-dev`

然后继续：

- `$finish-work`
- 人工测试并提交后，更新 `notes.md` 中的最终状态与交接信息

## 推荐流程

1. 阅读所提供的任务文档
2. 判断其属于：
   - 可直接执行
   - 简单 PRD
   - 复杂 PRD
3. 若将开始执行，运行 `before-dev`
4. 实现
5. 运行 `check-dev`
6. 运行 `finish-work`
