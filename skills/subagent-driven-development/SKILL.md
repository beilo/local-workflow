# 子代理驱动开发

这是可选的执行类技能。

不属于默认工作流。

仅当以下情况使用：

- 用户明确要求子代理执行，或
- 任务可清晰拆成相互独立的块

## 可接受的输入

本技能可从以下任一文档执行：

- `plan.md`
- `prd.md`
- 用户明确拖入或指向的任务文档

## 文档处理规则

### 若文档为 `plan.md`

直接使用。

### 若文档为简单的 `prd.md`

优先不用子代理。

简单 PRD 通常应留在主会话，使用 `executing-plans` 或直接实现。

### 若文档为结构化但复杂的 `prd.md`

可先拆成明确子任务，再分派子代理。

若仍过于模糊，先停并转为 `plan.md`。

## 前后规则

任何子代理开始写代码之前：

- 控制方必须先做等价于 `$before-dev` 的步骤
- 每个子代理须收到相关任务上下文与 spec 路径

代码完成后：

- 对整个任务做 `check-dev`
- 然后继续 `finish-work`
- 人工测试并提交后再更新 `notes.md` 中的最终状态与交接信息

## 推荐流程

1. 阅读所提供的任务文档
2. 判断是否适合子代理执行
3. 若适合，执行 `before-dev`
4. 分派子代理，附带：
   - 任务范围
   - 涉及文件
   - 相关 spec 文件
5. 整合结果
6. 对整个任务运行 `check-dev`
7. 运行 `finish-work`
