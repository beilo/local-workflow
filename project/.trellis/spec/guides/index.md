# 思考指南

> **目的**：拓宽思路，捕捉你可能尚未考虑到的方面。

---

## 为什么需要思考指南？

**大多数缺陷与技术债来自「没想到」**，而非能力不足：

- 没想到分层边界会发生什么 → 跨层缺陷
- 没想到代码模式会重复 → 到处复制粘贴
- 没想到边界情况 → 运行时错误
- 没想到未来维护者 → 代码难以理解

这些指南帮助你在**写代码之前先问对问题**。

## 速查：思考触发条件

### 何时考虑跨层问题

- [ ] 功能触及 3 层以上（API、Service、组件、数据库）
- [ ] 数据在各层之间格式发生变化
- [ ] 多个消费者需要同一份数据
- [ ] 不确定某段逻辑应放在哪一层

→ 阅读 [跨层思考指南](./cross-layer-thinking-guide.md)

### 何时考虑代码复用

- [ ] 正在写与已有代码相似的实现
- [ ] 同一模式重复出现 3 次以上
- [ ] 正在向多处添加新字段
- [ ] **正在修改任意常量或配置**
- [ ] **正在新建工具/辅助函数** ← 先搜索！

→ 阅读 [代码复用思考指南](./code-reuse-thinking-guide.md)

## 修改前规则（关键）

> **在修改任意取值之前，务必先搜索！**

```bash
# 搜索你即将修改的值
grep -r "value_to_change" .
```

这一个习惯能避免大多数「忘了改某处」类缺陷。
